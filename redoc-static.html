Collect sensor measurements

Allows for the submission of environmental sensor data, particularly CO2 concentration measurements. Sensors identified by their UUID can post readings, enabling real-time environmental monitoring and data collection for analysis. This endpoint is critical for aggregating sensor data across various locations, facilitating environmental assessments and responsive actions based on CO2 levels.

Authorizations:
bearerAuth
path Parameters
uuid
required
string

The unique identifier of the sensor from which the measurements originate. This UUID is essential for correctly associating the incoming data with the specific sensor in the system's database.

Request Body schema: application/json
required
co2
integer <int32>

CO2 concentration measured by the sensor, specified in parts per million (ppm). This measurement is vital for tracking air quality and identifying potential environmental concerns.

time
string <date-time>

The timestamp marking when the measurement was taken, according to ISO 8601 format. This precision ensures that each data point can be accurately placed within a temporal context for analysis.

Responses

The production API server.

The reference API server.

The acceptance API server.

http://acc.weather.com:{port}/{basePath}/api/v1/sensors/{uuid}/measurements

Get sensor status

Retrieves the current operational status of a specified sensor, identified by its unique UUID. This endpoint is pivotal for monitoring sensor health, enabling immediate response to any identified issues. It categorizes sensor status into three distinct levels: OK, WARN, and ALERT, each indicating varying degrees of operational normalcy and urgency.

Authorizations:
bearerAuth
path Parameters
uuid
required
string

The unique identifier (UUID) of the sensor whose status is being queried.

Responses

Local mock API server.

http://127.0.0.1:{port}/api/v1/sensors/{uuid}

The production API server.

http://weather.com:{port}/{basePath}/api/v1/sensors/{uuid}

The reference API server.

http://ref.weather:{port}/{basePath}/api/v1/sensors/{uuid}

The acceptance API server.

http://acc.weather.com:{port}/{basePath}/api/v1/sensors/{uuid}

Get sensor metrics

Retrieves comprehensive metrics for a specific sensor, identified by its unique UUID. This endpoint is designed to provide valuable statistical insights into environmental data captured by the sensor, such as CO2 levels over the past 30 days. It's instrumental for monitoring air quality trends and assessing environmental conditions in real-time.

List all alerts for a specific sensor

Retrieves a list of alerts generated by the specified sensor. This endpoint is crucial for monitoring sensor health and activity by providing a detailed view of all alerts associated with a sensor's unique identifier (UUID). It facilitates timely responses to potential issues and supports maintenance and diagnostic processes.
